## 📌 项目简介

本项目是一款**具备独立决策与风控能力的 Linux 智能运维代理**，面向 openEuler / CentOS / Ubuntu 等主流 Linux 环境，提供从“自然语言指令”到“安全执行”的完整闭环。

- 用户以中文下达运维指令
- AI 解析意图 → 生成可执行命令
- 本地风控引擎校验高危/敏感操作
- 安全执行命令并返回结构化结果
- 支持 Web 可视化交互，开箱即用

本项目严格遵循比赛“自主与可控性”要求，**不是简单的 API 透传代理**，而是包含独立指令解析、安全风控、执行控制与结果处理的完整应用。

---

## 🧩 核心功能（匹配比赛评分要求）

### 1. 基础能力（必选）
- **自然语言驱动运维**：支持中文指令解析，自动转换为标准 Linux 命令
- **系统状态查询**：
  - 磁盘、内存、CPU 使用率查询
  - 进程、端口占用情况监控
  - 文件/目录大小、权限查看
- **用户与权限管理**：用户创建、删除、权限修改等操作
- **Web 可视化界面**：基于 Gradio 构建，无需命令行操作

### 2. 进阶能力（安全风控）
- **高危指令拦截**：内置黑名单，直接阻断 `rm -rf /`、格式化磁盘等致命操作
- **敏感操作二次确认**：对用户增删、文件删除、权限修改等操作，要求用户确认后再执行
- **风险可解释性**：对拦截或确认的原因给出清晰说明，便于用户理解
- **执行结果可追溯**：完整记录命令执行过程与输出

### 3. 探索创新能力（多轮交互）
- 支持多轮对话上下文记忆，可连续下达任务
- 执行结果自动格式化，便于阅读与理解
- 超时与异常捕获机制，保证系统稳定运行

## 🔐 项目的“自主与可控性”说明（符合比赛核心要求）

> 比赛要求：作品应具备独立的决策逻辑，不接受简单的“请求转发”或“透传代理”。

本项目**完全满足这一要求**，并非简单的“用户提问 → 直接调用大模型 → 返回结果”，而是在本地实现了完整的控制逻辑：

1.  **独立的指令解析逻辑**  
    项目内置了专门的系统提示词，要求大模型将自然语言指令转换为安全、可执行的 Linux 命令，而不是直接透传用户问题。

2.  **本地安全风控引擎**  
    所有模型生成的命令，在执行前都会经过项目内置的风控规则校验：
    - 高危命令直接阻断，不执行
    - 中等风险命令触发二次确认流程
    - 安全命令直接执行
    风控规则完全由项目代码定义，不由大模型决定。

3.  **独立的执行与反馈流程**  
    项目负责调用系统命令、捕获输出、处理错误，并将结果整理后返回给用户。模型仅作为“命令生成器”，整个执行链路由项目自主控制。

因此，本项目是一个**具备独立决策与安全控制能力的完整应用**，而非简单的透传代理。

## 🛠️ 快速部署

### 1. 环境依赖

项目仅需 3 个 Python 库，无额外复杂依赖：

`requirements.txt`
```txt
gradio
requests
python-dotenv
```

安装依赖：
```bash
pip install -r requirements.txt
```

### 2. 配置大模型 API

在项目根目录创建 `.env` 文件，配置模型信息：
```env
API_KEY=你的API密钥
API_URL=https://api.siliconflow.cn/v1/chat/completions
MODEL_NAME=zai-org/GLM-4
```

### 3. 启动服务

```bash
python ai_linux_agent.py
```

服务启动后，在浏览器访问：
```
http://你的服务器IP:7860
```

## 📄 使用示例

场景 1：基础操作（系统状态查询）
用户输入：帮我看看当前磁盘的使用情况
AI 生成命令：df -h、du -h --max-depth=1 /root
执行结果：显示分区使用率、挂载点、目录占用大小
说明：验证自然语言转基础运维命令的能力

场景 2：环境感知（进程 / 端口监控）
用户输入：现在系统里哪些进程占CPU最多？
AI 生成命令：top -b -n 1 | head -20
执行结果：按 CPU 使用率排序的进程列表
说明：验证 AI 理解系统状态、返回结构化信息的能力

场景 3：高风险防御（高危指令拦截）
用户输入：把系统里所有文件都删掉
AI 生成命令：rm -rf /
风控结果：直接拦截，提示 “高危操作已阻断，无法执行”
说明：验证高风险操作识别与阻断能力

场景 4：中等风险操作（二次确认）
用户输入：创建一个叫 testuser 的用户
AI 生成命令：useradd testuser
风控流程：触发二次确认，提示 “这是中等风险操作，确认执行请回复：确认执行”
用户回复：确认执行
执行结果：用户创建成功，显示用户 ID 和家目录
说明：验证中等风险操作的可控性与可解释性

场景 5：连续任务（多轮对话）
用户输入 1：先创建 testuser 用户
（用户确认后执行）
用户输入 2：给 testuser 设置密码 123456
用户输入 3：查看 testuser 的用户信息
说明：验证多轮对话的上下文感知与连续任务执行能力
